Common Pitfalls and Best Practices

1. Dangling Pointers âŒ 

int* getDanglingPointer() {
    int x = 10;
    return &x;  // âŒ DANGER! x is destroyed after function returns
}

int main() {
    int *ptr = getDanglingPointer();
    printf("%d", *ptr);  // Undefined behavior!
    return 0;
}

Solution: Use static or dynamic allocation

int* getSafePointer() {
    static int x = 10;  // âœ… Persists after function returns
    return &x;
}

2. Uninitialized Pointers (Wild Pointers) âŒ 

int *ptr;        // Wild pointer (points to random memory)
*ptr = 10;       // âŒ CRASH! Writing to random memory

Solution: Always initialize pointers

int *ptr = NULL;     // âœ… Initialize to NULL
// OR
int x = 10;
int *ptr = &x;       // âœ… Initialize to valid address

3. Memory Leaks (C++ Dynamic Allocation) âŒ 

void memoryLeak() {
    int *ptr = new int[1000];  // Allocate memory
    // Forgot to delete!
}  // Memory leaked!

Solution: Always free/delete allocated memory

void noMemoryLeak() {
    int *ptr = new int[1000];
    // Use ptr...
    delete[] ptr;  // âœ… Free memory
}

4. Array Index Out of Bounds âŒ 

int arr[5] = {1, 2, 3, 4, 5};
cout << arr[10];  // âŒ Undefined behavior!

Solution: Check array bounds

int index = 10;
if(index >= 0 && index < 5) {
    cout << arr[index];  // âœ… Safe
}

5. Confusing Array of Pointers with Pointer to Array âŒ 

int *arr[5];        // Array of 5 pointers to int
int (*ptr)[5];      // Pointer to array of 5 ints

// They are NOT the same!

6. Modifying String Literals âŒ 

char *str = "hello";
str[0] = 'H';  // âŒ Undefined behavior (may crash)

Solution: Use character array for modifiable strings

char str[] = "hello";
str[0] = 'H';  // âœ… Safe (modifiable array)

Best Practices Summary

âœ… DO:

  • Always initialize pointers before use
  • Check for NULL before dereferencing
  • Use const to protect data that shouldn't be modified
  • Free dynamically allocated memory
  • Use meaningful variable names
  • Check array bounds
  • Prefer pass by address/reference for large structs

âŒ DON'T:

  • Return addresses of local variables
  • Dereference uninitialized pointers
  • Modify string literals
  • Mix pointer types without explicit casting
  • Forget to free allocated memory
  • Access array out of bounds
  • Use pointer arithmetic carelessly

    int temp = x;
    x = y;
    y = temp;
    // Only swaps local copies!
}

// CORRECT: Pass by address (swaps originals)
void swap_correct(int *x, int *y) {
    int temp = *x;
    *x = *y;
    *y = temp;
}

int main() {

    printf("After swap_wrong: a=%d, b=%d\n", a, b);
    // Output: a=5, b=10 (unchanged)
    
    swap_correct(&a, &b);
    printf("After swap_correct: a=%d, b=%d\n", a, b);
    // Output: a=10, b=5 (swapped!)
    
    return 0;
}

Summary

Key Concepts

  1. Pointers store memory addresses and provide powerful ways to manipulate data
  2. Scope determines where a variable can be accessed
  3. Lifetime determines how long a variable exists in memory
  4. Static variables persist for the entire program but retain local scope
  5. Pass by address allows functions to modify original variables
  6. Const protects data from unwanted modification
  7. Structs group related data of different types
  8. Arrays and pointers are closely related but not identical

Common Use Cases

  • Pointers: Dynamic memory, efficient parameter passing, data structures
  • Static variables: Counters, caching, persistent state within functions
  • Const pointers: Function parameters that shouldn't modify data
  • Structs: Organizing related data (students, employees, dates)
  • Arrays of pointers: String arrays, dynamic 2D arrays
  • Pointer to array: 2D array manipulation

Remember

âœ… Always:

  • Initialize pointers before use
  • Check for NULL before dereferencing
  • Match types correctly
  • Free dynamically allocated memory
  • Use const for protection

âŒ Never:

  • Return addresses of local variables (unless static)
  • Dereference uninitialized pointers
  • Modify string literals
  • Access arrays out of bounds
  • Forget to free allocated memory
